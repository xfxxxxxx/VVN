# Virginia Arts Vibrancy Index — prototype

A DSPG × **SMU DataArts** prototype that adapts the national **Arts Vibrancy Index (AVI)** framework to
Virginia and presents it as an integrated data story: a statewide map plus a tight narrative, backed by a
fully reproducible raw → processed → index pipeline.

> ⚠️ **Synthetic prototype.** Every arts/economic figure here is **synthetic and illustrative** — built to
> demonstrate the pipeline and the data story, *not* to report real arts activity. County boundaries and
> populations are real. The pipeline is structured so a team can drop in live data (Census CBP/ZBP, IRS
> Form 990, NEA/IMLS via FOIA, NASAA) and rerun unchanged.

## The data story

`docs/index.html` is the deliverable webpage — visuals and narrative on one page:

- **Figure 1 — statewide choropleth** of all 133 Virginia localities, shaded by Arts Vibrancy (brighter =
  more vibrant). Hover any locality for its overall rank and its rank on each of the three dimensions.
- **Two-paragraph narrative** beside the map: where vibrancy concentrates, and how the composite is built.
- **Figure 2 — top-15 leaderboard** with per-dimension standing.
- **Method strip**: six measures → three dimensions → one equal-weighted score.

Open it locally (`open docs/index.html`) or publish it free via **GitHub Pages** (Settings → Pages →
deploy from `/docs`). The page is self-contained; the only external dependency is Leaflet from a CDN.

## Reproduce the results

```bash
# from the repo root
Rscript R/01_build_index.R
```

This reads `data/raw/arts_vibrancy_va_synthetic.csv` and writes
`data/processed/va_arts_vibrancy_index.csv`, printing the top 10. Anyone running the script against the
same raw file reproduces the identical processed dataset and rankings.

**Method (matches the AVI framework, equal-weighting variant):**
1. Impute the one suppressed value (`nonprofit_rev_k`, Charlottesville) from payroll — made explicit, not dropped.
2. Standardize the six arts measures (z-score).
3. Average within each dimension → `providers_z`, `dollars_z`, `grants_z`.
4. Equal-weight mean of all six z-scores → `avi_z`; rescale to 0–100; rank.

The three ACS context variables (income, % college, % poverty) are held out to **validate** the index, not
to build it. Equal weighting is the interpretable first pass; PCA / latent-factor weighting are natural
next steps (see the SMU DataArts AVI docs and the OECD/JRC Handbook on Composite Indicators).

> Note: R is not installed in the authoring sandbox, so the shipped `processed/` file was produced by the
> identical steps in Python; `Rscript R/01_build_index.R` regenerates it in the team's R environment.

## Data sources (for the real build)

| Dimension | Source | Access |
|---|---|---|
| Arts Providers | County/ZIP Business Patterns (Census); IRS 990 org counts | Census API · IRS bulk downloads |
| Arts Dollars | IRS Form 990 (revenue, expenses, compensation) | IRS bulk downloads |
| Grant Activity | NEA & IMLS (federal); NASAA (state) | FOIA · NASAA |
| Context (validation) | American Community Survey | Census API (`tidycensus`) |
| Adjustments | Per-capita; C2ER cost-of-living index (`colixp`) | C2ER |
